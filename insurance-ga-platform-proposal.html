01

프로젝트 개요 및 목표

보험 GA(General Agency)는 다수의 보험사 상품을 동시에 취급하기 때문에, 모집인이 각 보험사 상품의 보장 내용·보험료·특약 구조를 정확히 파악하고 고객에게 설명하는 데 상당한 시간과 노력이 소요됩니다. 본 플랫폼은 이 과정을 디지털 자동화하여 모집인 생산성을 높이고, 고객의 이해도를 극대화하는 데 목적이 있습니다.

🎯 문제 정의

보험사별 상이한 상품명·특약 구조로 인한 모집인 혼란 및 고객 설명 비효율

  • 보험사마다 다른 상품 명칭 및 구조
  • 보장 비교 시 수작업 엑셀 의존
  • 고객 맞춤 설명 자료 제작에 과다 시간 소요
  • 실시간 보험료 시뮬레이션 불가

✅ 솔루션 방향

단일 웹 플랫폼에서 모든 보험사 상품을 비교하고, 고객용 설명 자료를 자동 생성

  • 보험사 통합 비교 대시보드
  • 실시간 보험료 계산기
  • 고객 맞춤 설명 자료 자동 생성
  • 모바일 고객 공유 기능

👥 타깃 사용자

GA 운영자, 소속 보험 모집인, 그리고 상담 고객 3가지 역할로 구분

  • GA 관리자: 상품 데이터 관리, 모집인 계정 관리
  • 보험 모집인: 상품 비교, 고객 제안서 생성
  • 고객(View-only): 제안서 열람, 간편 상담 신청
💡 핵심 가치: 모집인이 "어떤 상품을 추천할까?" 고민하는 시간을 줄이고, 고객이 "이 상품이 나에게 맞는가?"를 쉽게 판단할 수 있도록 시각적·직관적 인터페이스를 제공합니다.
02

핵심 기능 정의

📊 보험상품 비교 엔진

최대 4개 상품을 나란히 비교하는 Side-by-Side 비교표. 보장 항목별 차이를 색상으로 강조 표시.

  • 보험사 / 상품명 / 보험 종류 필터
  • 보장 항목별 점수화 (★ 표시)
  • 항목별 유불리 자동 강조
  • PDF 비교표 다운로드

🧮 실시간 보험료 계산기

성별, 나이, 납입 기간, 보장 금액 등을 입력하면 복수 보험사의 보험료를 즉시 계산.

  • 인터랙티브 슬라이더 UI
  • 납입 방법별 (월납/연납) 비교
  • 예상 환급률 표시
  • 연령 변화에 따른 보험료 그래프

📋 고객 제안서 + 상담 신청 CTA

선택한 상품과 고객 정보를 조합해 브랜드화된 제안서를 자동으로 생성·공유. 각 상품 카드에 "이 상품으로 가입 상담 신청" 버튼으로 즉시 모집인 연결.

  • GA 브랜드 로고 적용
  • 고객 이름 개인화
  • 상품 카드별 "가입 상담 신청" CTA 버튼
  • 신청 즉시 모집인 카카오 알림톡 발송
  • 카카오톡/문자 링크 공유 · QR코드

🤖 AI 상품 추천 엔진 (LLM 연동)

고객 프로필을 LLM(GPT-4o / Claude / Gemini)에 전달해 최적 상품을 추천하고, 단순 템플릿이 아닌 자연어로 설득력 있는 추천 이유를 자동 생성. 고객 화면에 AI 추천임을 명확히 표시하여 신뢰도 향상.

  • 가중치 점수 계산 → LLM 자연어 해설 생성
  • 고객 제안서 상단 "AI 맞춤 분석" 배너 표시
  • 상품 카드별 "AI 추천 N위" 뱃지 + 종합점수
  • 보장·보험료·갱신 안정성 항목별 점수 바 표시
  • "AI 추천 이유" 블록 — LLM 생성 자연어 문장
  • 하단 AI 분석 면책 고지 (신뢰 보조 장치)

📱 모바일 고객 뷰

고객이 스마트폰으로 접근하는 전용 읽기 전용 뷰. 직관적 카드 UI로 상품 이해 지원.

  • 스와이프 비교 (모바일 최적화)
  • 보장 내용 아이콘화
  • 원터치 상담 신청 버튼
  • 만료 기간 설정 (보안)

⚙️ GA 관리자 패널

보험 상품 데이터 입력·수정, 모집인 계정 관리, 통계 대시보드.

  • 상품 데이터 CRUD
  • 엑셀 일괄 업로드
  • 모집인별 활동 현황
  • 제안서 열람 횟수 통계

기술 스택

영역 기술 선택 선택 이유
프론트엔드 Next.js 14 React 18 TypeScript App Router 기반 SSR/SSG 지원, SEO 최적화, Vercel 네이티브 배포. TypeScript로 보험 데이터 타입 안전성 확보.
스타일링 Tailwind CSS shadcn/ui 빠른 반응형 UI 개발, shadcn/ui로 접근성 있는 컴포넌트 기반 구성. 커스텀 GA 브랜드 테마 적용 용이.
차트/시각화 Chart.js Recharts 보험료 비교 바 차트, 환급률 라인 차트, 보장 레이더 차트에 활용. Recharts는 React 친화적 구성.
백엔드 Next.js API Routes Prisma ORM 별도 백엔드 서버 불필요. API Routes로 보험료 계산, 제안서 생성 로직 처리. Prisma로 타입 세이프 DB 쿼리.
데이터베이스 PostgreSQL Vercel Postgres 복잡한 보험 상품 구조(특약, 보장 항목)를 관계형으로 저장. Vercel Postgres로 배포 환경과 통합 관리.
인증 NextAuth.js JWT GA 관리자 / 모집인 / 고객 3가지 역할(Role) 기반 접근 제어. 카카오 소셜 로그인 연동 지원.
파일/이미지 Vercel Blob Sharp GA 로고, 보험사 로고 이미지 저장. Sharp로 제안서 PDF 생성 시 이미지 최적화.
PDF 생성 Puppeteer jsPDF 비교표 및 제안서 PDF 출력. Puppeteer는 서버사이드 고품질 PDF, jsPDF는 클라이언트 간편 출력에 활용.
상태 관리 Zustand React Query 비교 선택 상태, 계산기 입력 상태는 Zustand. 보험 데이터 서버 페칭·캐싱은 React Query(TanStack Query).
AI / LLM OpenAI GPT-4o Claude API Gemini Pro 추천 이유 자연어 생성·제안서 문구 자동 작성에 활용. GA 설정 화면에서 사용 모델 전환 가능. API 비용 절감을 위해 추천 결과 캐싱 전략 병행.
배포 Vercel GitHub Actions GitHub push → 자동 배포. PR별 Preview URL 생성. CI에서 TypeScript 타입 체크, ESLint, Vitest 자동 실행.

개발 단계 (Phase)

Phase 1 — 기반 구축 및 데이터 모델 1~2주차

프로젝트 초기 셋업, DB 설계, 인증 시스템 구축. 엑셀 업로드로 실제 보험 상품 데이터를 최초 적재하고, 개인정보 암호화·금소법 면책 고지를 기반 구축 단계에서 함께 완성.

Next.js 프로젝트 초기화 GitHub 레포 생성 Vercel 연동 Prisma 스키마 설계 NextAuth 설정 관리자/모집인 Role 설정 ⬆ 엑셀 일괄 업로드 (P3→P0 격상) ⬆ 개인정보 AES-256 암호화 (Phase 1 필수) ⬆ 금소법 면책 고지 Sticky Footer (전 화면)
⬆ [평가 반영] 엑셀 업로드를 Phase 4 → Phase 1으로 격상. 상품 데이터 없이는 비교·계산 기능 자체를 테스트할 수 없으므로 개발 초기에 반드시 선행. 금소법 면책 고지도 개발 첫날부터 전 화면 Sticky Footer로 적용.
Phase 2 — 핵심 비교 기능 개발 3~5주차

플랫폼의 핵심인 상품 비교 엔진과 보험료 계산기 개발. 계산기는 Phase 2에서 "연령·성별 기준 표준 가이드 보험료"만 제공하고, 직업 급수·병력 등 정밀 변수 반영은 Phase 4로 이연.

상품 목록·필터 API Side-by-Side 비교표 UI 보장 항목 하이라이트 로직 표준 가이드 보험료 계산 (연령·성별) 슬라이더 계산기 UI Chart.js 보험료 그래프 Zustand 비교 상태 관리 "상담으로 정확한 금액 확인" CTA 버튼
⬆ [평가 반영] 보험료는 직업 급수·과거 병력 등으로 계산 로직이 매우 복잡함. Phase 2에서는 표준 연령별 가이드 보험료를 "예상 금액" 으로 표시하고, 정밀 계산은 상담 연결로 유도하는 현실적 전략으로 수정.
Phase 3 — 제안서 시스템 개발 6~8주차

고객 제안서 생성·공유·열람 시스템 구현. 모바일 최적화 고객 뷰 완성. 고객이 링크를 클릭하는 순간 모집인에게 실시간 알림 발송. 이 플랫폼의 핵심 차별점인 LLM 기반 AI 추천 엔진을 Phase 3에서 조기 확보하여 경쟁력을 선점.

제안서 Builder UI 공유 토큰 생성 API 고객용 Share 페이지 PDF 생성 (Puppeteer) 카카오 공유 연동 QR코드 생성 제안서 열람 통계 ★ 열람 실시간 알림 (Vercel Edge + 카카오 알림톡) ★ 제안서 워터마크 (모집인명 삽입) 🤖 LLM AI 추천 엔진 (Phase 4→3 조기 확보) 🤖 가중치 점수 → LLM 자연어 추천 이유 생성 🤖 추천 결과 제안서 자동 삽입 📲 상품별 "가입 상담 신청" CTA 버튼 📲 상담 신청 모달 (연락처 입력) 📲 신청 즉시 모집인 카카오 알림톡 발송 📲 상담 신청 완료 화면 (담당 모집인 정보) 📲 30분 이내 신청 취소 기능
🤖 [AI 추천 Phase 4→3 조기 확보] AI 추천이 이 플랫폼의 핵심 차별점이라면 Phase 3에서 선제적으로 구현하는 것이 전략적으로 유리함. 고객 프로필 입력 → 가중치 점수 계산 → LLM API 호출 → 자연어 추천 이유 생성 → 제안서 자동 삽입 파이프라인. LLM 모델은 llm-client.ts 추상화 레이어로 GPT-4o / Claude / Gemini 중 GA 설정에서 선택 가능.
Phase 4 — 관리자 패널 및 고도화 9~12주차

GA 관리자용 상품 관리·통계 패널 완성, 직업 급수·병력 등 변수를 반영한 정밀 보험료 계산 엔진 고도화, LLM 추천 결과 품질 개선 및 성능 최적화.

관리자 상품 CRUD 모집인 활동 대시보드 정밀 보험료 계산 고도화 (직업급수·병력) LLM 추천 품질 개선 (프롬프트 튜닝) 추천 결과 캐싱 (API 비용 절감) 성능 최적화 (캐싱) 다크 모드 (디자인 QA 완료 후) 최종 QA 및 배포

GA 채널 통제 시스템

모집인이 GA 시스템을 이용하면서 고객을 임의로 타 GA에 연결하거나 GA 승인 없이 독자 계약을 진행하는 것을 방지하기 위한 4개 레이어 통제 구조입니다. 기술적 제한과 경제적 유인을 함께 설계하여 모집인이 자발적으로 시스템 안에서 영업하도록 유도합니다.
⚠️ 도입 전략: Layer 1·2는 Phase 1~3에서 즉시 적용. Layer 3(모니터링)은 초기에는 내부용으로만 운영하다가 필요 시 공개. 지나친 통제는 모집인 반발을 유발할 수 있으므로 단계적으로 도입.

Layer 1 — 고객 데이터 소유권 통제

고객 정보는 GA 소유로 DB에 귀속. 모집인은 열람 권한만 가지며 소유권이 없습니다. 퇴사·이직 시 계정 비활성화만으로 모든 접근이 즉시 차단됩니다.

고객 DB 귀속 구조

고객 레코드에 ownedByGaId 필드 고정. 모집인 FK는 담당자 표시용일 뿐, 소유권과 분리.

  • 모집인 계정 삭제 시 고객 데이터는 GA에 유지
  • 담당 모집인 재배정 기능 (GA 관리자 전용)

고객 목록 내보내기 차단

모집인 Role은 고객 1건씩 조회만 가능. 전체 목록 CSV/Excel 다운로드는 GA 관리자만 허용.

  • Export API — AGENT Role 접근 시 403 차단
  • 화면 내 복사 방지 (선택적 적용)

Layer 2 — 제안서·계약 흐름 통제

제안서 발송부터 계약 완료까지 GA 시스템 안에서만 진행되도록 워크플로우를 설계합니다. 시스템 외부로 계약이 빠져나가는 경로를 구조적으로 차단.

제안서 GA 브랜딩 고정

모든 제안서에 GA 로고·연락처·고유 식별 코드를 서버사이드에서 강제 삽입. 모집인이 임의 수정 불가.

  • PDF 생성 시 GA 브랜드 레이어 오버레이
  • 공유 URL에 GA 도메인 고정

계약 완료 신고 의무화

고객이 제안서 열람 후 3일 이내 "계약 완료 등록"이 없으면 GA 관리자에게 자동 알림. 누수 계약 조기 감지.

  • ProposalStatus: viewed → pending → completed
  • 3일 경과 시 관리자 알림 자동 발송

고액 계약 관리자 승인

월 보험료 기준액 이상 제안서는 GA 관리자 승인 후 발송. 대형 건을 GA 모르게 독자 처리하는 것 방지.

  • 임계값은 GA 설정 화면에서 조정 가능
  • 승인 대기 중 제안서는 발송 잠금

Layer 3 — 이상 행동 감지 (내부 모니터링)

시스템 사용 패턴을 분석해 GA 이탈 징후를 조기에 포착합니다. 모집인에게는 노출하지 않고 GA 관리자 대시보드에서만 확인하는 내부 운영 도구입니다.

계약 전환율 추적

모집인별 "제안서 발송 수 대비 계약 완료 수" 비율. 전환율 급락 = 외부 채널 유출 신호.

  • GA 관리자 대시보드 전용 차트
  • 월별 트렌드 비교

시스템 비활동 감지

7일 이상 제안서 미생성 모집인을 GA 관리자에게 알림. 시스템 우회 영업 가능성 감지.

  • Vercel Cron Job으로 매일 자동 점검
  • 비활동 기간 설정 가능

접속 패턴 로그

새벽 대량 조회, 단시간 다수 고객 접근 등 비정상 패턴 기록. 주간 리포트로 관리자에게 발송.

  • ActivityLog 테이블 별도 관리
  • 이상 패턴 감지 쿼리 자동 실행

Layer 4 — 수수료 연동 잠금 (가장 강력한 통제)

"시스템 밖 계약 = 수수료 없음" 구조를 만들면 모집인이 스스로 시스템을 사용하게 됩니다. 기술적 강제보다 경제적 유인이 훨씬 강력하고 지속 가능한 통제 수단입니다.

수수료 정산 시스템 연계

Commission 테이블에 proposalId FK 필수. 시스템에 등록된 계약 건에만 수수료 정산 가능.

  • 모집인별 정산 현황 대시보드
  • 미연결 계약 건 정산 API 차단

모집인 이용 약관 전자 서명

계정 생성 시 "GA 시스템 통한 영업 의무, 고객 데이터 GA 귀속" 전자 서명. 분쟁 시 법적 근거.

  • 서명일시·IP·UserAgent 기록
  • 약관 변경 시 재동의 요청

LLM AI 추천 엔진 상세 설계

기존 "항목 가중치 기반 점수 계산"은 유지하되, 그 결과를 LLM에게 전달해 자연어로 설득력 있는 추천 이유를 생성하는 2단계 파이프라인입니다. 단순 템플릿 대비 고객별 맞춤 표현이 가능하여 제안서의 전문성과 설득력이 크게 향상됩니다.

AI 추천 파이프라인 (2단계)

Step 1 — 가중치 점수 계산 (서버 로직)

고객 프로필 입력값을 기반으로 각 보험 상품에 수치 점수를 부여.

  • 나이·성별·직업·건강 상태·예산 입력
  • 보장 항목별 가중치 × 상품 점수 합산
  • 상위 3개 상품 선정
  • 점수 계산은 DB 쿼리만으로 처리 (LLM 미사용, 빠름)

Step 2 — LLM 자연어 생성 (API 호출)

점수 계산 결과와 고객 프로필을 LLM에 전달해 추천 이유 문장을 생성.

  • 시스템 프롬프트: "당신은 10년 경력의 보험 전문 FP입니다"
  • 고객 정보 + 상위 3개 상품 데이터 → LLM 입력
  • 출력: 상품별 2~3문장 추천 이유 (자연어)
  • 결과 캐싱 (동일 프로필 재호출 시 API 비용 절감)

LLM 추상화 레이어 설계 (llm-client.ts)

상품 연결 상담 신청 UI (방법 B)

고객이 제안서에서 마음에 드는 상품을 발견했을 때 그 자리에서 바로 가입 상담을 신청할 수 있는 UI입니다. 신청 즉시 담당 모집인에게 카카오 알림톡이 발송되어 빠른 팔로업이 가능하고, GA 시스템 안에서 계약 흐름이 완결되어 채널 통제 원칙도 유지됩니다.

AI 추천 표시 UI 구성 요소 (고객 화면)

① AI 맞춤 분석 배너 (최상단)

제안서 진입 직후 가장 먼저 보이는 위치에 퍼플 배너로 AI 분석 사실을 알림. 고객이 "왜 이 상품이 나에게 추천됐는지" 맥락을 먼저 이해하게 함.

  • 문구: "AI가 고객님 맞춤 분석으로 추천한 상품입니다"
  • 부제: "나이·직업·건강 상태·예산을 종합 분석하였습니다"
  • AI 아이콘 + 퍼플 배경으로 시각적 구분

② 상품 카드 — AI 추천 뱃지 + 점수

각 상품 카드 우측 상단에 "AI 추천 1위 / 2위 / 3위" 뱃지와 종합점수(100점 만점) 표시. 1위 카드는 테두리 강조로 시각적 우선순위 명확화.

  • AI 추천 순위 뱃지 (퍼플 계열)
  • 종합점수 숫자 표시
  • 1위 카드: 보더 2px 강조

③ 항목별 점수 바

보장 범위·보험료 적정성·갱신 안정성 3개 항목을 점수 바로 시각화. 고객이 수치로 상품을 직관적으로 비교할 수 있음.

  • 보장 범위 / 보험료 적정성 / 갱신 안정성
  • 퍼플 채움 바 + 우측 숫자
  • 상품 간 한눈에 비교 가능

④ AI 추천 이유 블록

각 상품 카드 내부에 LLM이 생성한 고객 맞춤 추천 이유 2~3문장. 퍼플 배경 블록으로 강조하여 일반 설명과 구분.

  • AI 아이콘 + "AI 추천 이유" 라벨
  • LLM 생성 자연어 2~3문장
  • 고객 직업·나이·예산을 맞춤 언급

⑤ 하단 AI 면책 고지

신뢰를 높이면서도 법적 리스크를 방지하는 소형 안내 문구. 방패 아이콘과 함께 회색 배경으로 눈에 띄지 않지만 존재감 있게 배치.

  • "AI 추천 결과는 참고 자료이며 최종 보험료는 상담으로 확정"
  • 방패 아이콘 + 작은 회색 텍스트
  • 금소법 면책 고지와 통합 배치
💡 디자인 원칙: AI 관련 모든 UI 요소는 퍼플(#534AB7) 계열로 통일합니다. 고객이 "보라색 = AI가 분석한 것"을 자연스럽게 연상하도록 색상 언어를 일관되게 유지하는 것이 핵심입니다. 1위 추천 상품은 퍼플 2px 테두리로 다른 상품과 명확히 구분합니다.

UI 상세 설계

① 상품 카드 CTA 버튼

고객 제안서의 각 상품 카드 하단에 위치. AI 추천 이유 문장 바로 아래 배치하여 설득 → 행동 전환을 자연스럽게 유도.

  • 버튼 텍스트: "이 상품으로 가입 상담 신청하기"
  • GA 브랜드 컬러 강조 버튼 (풀 width)
  • 이미 신청한 상품은 "신청 완료" 상태로 비활성화
  • 복수 상품 동시 신청 가능

② 연락처 입력 Bottom Sheet

버튼 클릭 시 화면 하단에서 올라오는 모달. 모바일 친화적 Bottom Sheet UI로 입력 편의성 극대화.

  • 신청 상품명 상단 확인 표시
  • 이름·연락처 입력 (기존 입력 정보 자동 채움)
  • 희망 연락 시간 선택 (오전/오후/저녁)
  • 개인정보 수집 동의 체크박스

③ 신청 완료 화면

신청 처리 후 고객에게 다음 단계를 명확히 안내. 모집인 정보를 노출해 신뢰감 제공.

  • 담당 모집인 이름·프로필 사진 표시
  • 예상 연락 시간 안내 (영업일 기준)
  • 카카오채널 바로가기 버튼
  • 30분 이내 신청 취소 버튼
